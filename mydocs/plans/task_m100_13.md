---

## 1. 목표

`PopgaCrawler`가 **라이브 popga.co.kr**에서 서브컬처 팝업을 정상 분류하도록 category 추출 로직을 수정한다.
현재 라이브 환경에서 서브컬처 분류 결과가 **0건**이며, 이로 인해 수집 파이프라인이 서브컬처 행사를 저장하지 못한다.

---

## 2. 현황 진단

### 2-1. 증상
- 최근(<=30일) 팝업 150건을 상세 파싱했을 때 **서브컬처 0건** (`isSubculture()` 전부 false).
- 상세 파싱 자체(title·시작일·종료일·장소·요약)는 정상 동작.

### 2-2. 근본 원인

| 단계 | 현재 동작 | 문제 |
|------|-----------|------|
| `parseDetail` | `parseNextData(html) ?? parseHtml(html)` | 라이브 페이지에 `__NEXT_DATA__`가 **없음** → 항상 `parseHtml` 폴백 |
| `parseHtml` (line 192) | category를 `[class*="category"], [class*="tag"], [class*="badge"]`에서 추출 | 현재 사이트 DOM 구조와 불일치 → **`category` 항상 빈 문자열** |
| `isSubculture('')` | `애니/캐릭터/게임` 포함 검사 | 빈 문자열이라 항상 false |

### 2-3. 실제 category 신호 위치
- 상세 페이지 `<meta name="keywords">`에 category·서브컬처 신호가 존재.
  예) `영등포,영등포 팝업,영등포 타임스퀘어,리빙,...` / `홍대,...,캐릭터,애니메이션,굿즈,...`
- meta keywords 기반으로 보정 시 동일 표본에서 **서브컬처 ~38건** 검출
  (귀멸의 칼날, 매지컬 치이카와, 산리오캐릭터즈, 디지몬 어드벤처, 토이스토리5, 블랙핑크 X 다마고치 등).

### 2-4. 테스트가 버그를 못 잡은 이유
- `PopgaCrawler.test.ts`는 `__NEXT_DATA__` fixture 기반 → `parseNextData` 경로만 검증.
- 라이브의 `parseHtml` 폴백 경로(및 meta keywords)는 미검증.

---

## 3. 변경 방안

### 3-1. `parseHtml` category 추출을 meta keywords 기반으로 보강
**변경 파일**: `backend/crawler/src/crawlers/PopgaCrawler.ts`

- `<meta name="keywords">`(보조로 `og:title`)에서 category 토큰/서브컬처 신호를 추출.
- 기존 `[class*=...]` 선택자는 보조 폴백으로 유지(향후 DOM 변경 대비).

### 3-2. `isSubculture` 판정 소스 확장
- 판정 입력을 category 단일 문자열 → **keywords + title + summary** 결합 텍스트로 확장.
- 서브컬처 키워드 집합 보강: `애니, 캐릭터, 게임, 애니메이션, 웹툰`(필요 시 조정).
- 단, 과탐(false positive) 방지를 위해 키워드 집합은 구현 계획서에서 확정.

### 3-3. 실 사이트 HTML 기반 회귀 테스트 추가
**변경 파일**: `backend/crawler/src/__tests__/PopgaCrawler.test.ts` (+ fixture 추가)
- `__NEXT_DATA__`가 없는 실제 구조의 HTML fixture 추가.
- meta keywords 기반 category/서브컬처 검출 케이스(양성/음성) 검증.
- 기존 `__NEXT_DATA__` 케이스는 유지(회귀 방지).
