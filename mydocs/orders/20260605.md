### Step 0. 사전 준비 (최초 1회)

```bash
# AWS CLI 자격증명 확인
aws sts get-caller-identity

# 의존성 설치 (각 패키지 개별 실행)
cd infra && npm install && cd ..
cd backend/api && npm install && cd ../..
cd backend/crawler && npm install && cd ../..
cd frontend && npm install && cd ..

# CDK 부트스트랩 (계정 최초 1회)
cd infra
npx cdk bootstrap aws://ACCOUNT_ID/ap-northeast-2
```

### [dev 배포] ~$29/월

#### Step 1. 변경 사항 미리보기

```bash
cd infra
npx cdk diff --all --context env=dev
```

#### Step 2. 전체 스택 배포

```bash
cd infra
npx cdk deploy --all --context env=dev
```

| 스택 | 주요 리소스 | 비용 |
|------|------------|------|
| `SubcultureTracker-Data-dev` | VPC + NAT Instance(**t3.nano**, AL2023) + RDS(**db.t3.micro**, 20GB) + SSM + DynamoDB | ~$26/월 |
| `SubcultureTracker-Api-dev` | Lambda API × 7 + API Gateway REST (로그 **7일**) | ~$1/월 |
| `SubcultureTracker-Crawler-dev` | Lambda Crawler + SQS + EventBridge (로그 **7일**) | ~$0/월 |
| `SubcultureTracker-Cert-dev` | ACM 인증서 (us-east-1) | 무료 |
| `SubcultureTracker-Frontend-dev` | S3 + CloudFront | ~$1/월 |

#### Step 3. 자격증명 SSM 수동 입력 (배포 후 필수)

```bash
aws ssm put-parameter \
  --name "/subculture-tracker/dev/naver-api" \
  --type "SecureString" \
  --value '{"clientId":"YOUR_ID","clientSecret":"YOUR_SECRET"}' \
  --overwrite

aws ssm put-parameter \
  --name "/subculture-tracker/dev/ncp-api" \
  --type "SecureString" \
  --value '{"clientId":"YOUR_ID","clientSecret":"YOUR_SECRET"}' \
  --overwrite
```

#### Step 4. 특정 스택만 재배포

```bash
cd infra
npx cdk deploy SubcultureTracker-Data-dev --context env=dev
npx cdk deploy SubcultureTracker-Api-dev --context env=dev
npx cdk deploy SubcultureTracker-Crawler-dev --context env=dev
npx cdk deploy SubcultureTracker-Frontend-dev --context env=dev
```

#### Step 5. 리소스 전체 삭제

```bash
cd infra
npx cdk destroy --all --context env=dev
# dev: RemovalPolicy.DESTROY — RDS·DynamoDB 데이터 함께 삭제됨
```
