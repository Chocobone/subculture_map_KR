#### 2-1. 화면 구성 (거지맵 구조 참고)

| 탭 | 거지맵 대응 | 우리 앱 화면 |
|----|------------|------------|
| 지도 | 지도 (/) | 팝업스토어 핀 지도 — Naver Maps, IP 카테고리별 핀 색상 |
| 목록 | 거지방 (/bang) | 행사 피드 — 카드 목록, 상태(진행중/예정/종료) 배지 |
| 신규 | 핫딜 (/deal) | 새로 등록된 팝업 피드 — 최신순, 종료 임박 강조 |
| MY | — | 관심 IP 관리, 구독 알림 설정 |

#### 2-2. 공통 컴포넌트

- **하단 탭바**: 지도 · 목록 · 신규 · MY (4탭, 아이콘 + 라벨)
- **행사 카드**: IP 썸네일, 제목, 기간(D-day), 장소명, 타입 배지(팝업/콜라보/굿즈/한정)
- **IP 필터 칩**: 수평 스크롤 칩 목록 (전체 + 등록된 IP)
- **상태 배지**: 진행중(초록) / 예정(파랑) / 종료(회색)
- **지도 핀 팝업**: 행사명, 기간, 장소명, 상세 이동 버튼

### 3. 로컬 테스트

**AWS 의존 항목 → Mock 처리 목록**

| AWS 서비스 | 의존 위치 | 로컬 대체 방법 |
|-----------|----------|--------------|
| Aurora PostgreSQL | `db/client.ts` | Docker `postgres:15` + 환경변수 `DB_PASSWORD` |
| DynamoDB | `rawStorage.ts` | DynamoDB Local (Docker) + `DYNAMO_ENDPOINT` |
| AWS Bedrock (Claude Haiku) | `BedrockClassifier.ts` | `MockBedrockClassifier` — 고정 샘플 응답 반환 |
| Naver Local Search API | `naverMapsService.ts` | `MockNaverMapsService` — 고정 좌표 반환 |
| Secrets Manager | `db/client.ts`, `naverMapsService.ts` | 환경변수 직접 주입 (`DB_PASSWORD`, `NAVER_CLIENT_ID`) |
| API Gateway + Lambda | `backend/api/` 핸들러 | SAM local (`sam local start-api --env-vars env.json`) |
| Cognito | `frontend/` 인증 | Mock JWT 토큰 (로컬 bypass) |
| SQS 크롤러 트리거 | `handler.ts` | 직접 invoke (`sam local invoke`) |
| SES/SNS 알림 | `notifier/` | Mock 함수 — 콘솔 출력으로 대체 |
| EventBridge 스케줄 | `CrawlerStack` | `sam local invoke` 수동 트리거 |

**로컬 테스트 실행 순서**

```bash
# 1. 인프라 (Docker)
docker run -d -e POSTGRES_PASSWORD=dev -p 5432:5432 postgres:15
docker run -d -p 8000:8000 amazon/dynamodb-local

# 2. DB 초기화
psql -h localhost -U postgres -d postgres -f backend/api/src/db/migrations/001_initial_schema.sql
psql -h localhost -U postgres -d postgres -f backend/api/src/db/migrations/002_add_source_url_unique.sql

# 3. API Lambda 로컬 실행
cd backend && sam local start-api --env-vars env.json

# 4. 프론트엔드 개발 서버
cd frontend && npm run dev   # localhost:5173

# 5. 크롤러 단위 테스트
cd backend/crawler && npm test
```
